# Project Rules

## Non-Open Source Project
This project is **NOT** open source. Do not:
- Add open source licenses
- Include public repository references
- Apply open source contribution guidelines
- Use open source collaboration workflows

## Code Standards
- Follow existing code style and conventions
- Maintain consistency with the codebase
- Keep files under 1000 lines maximum
- Avoid god classes
- Prefer composition over inheritance
- Ensure integration test coverage for cross-system integrations
