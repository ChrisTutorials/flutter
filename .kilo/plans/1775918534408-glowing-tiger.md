# Unit Converter UI Visual Design Audit & Remediation Plan

## Audit Summary
Comprehensive audit identified **17 issues** across **4 severity levels** affecting visual design quality and Material 3 compliance.

| Severity | Count | Impact |
|----------|-------|--------|
| CRITICAL | 4 | Theme toggle broken, hardcoded colors break dark/light and palette switching |
| HIGH | 4 | Card inconsistencies, button mixing, breakpoint mismatches |
| MEDIUM | 5 | Responsive layout gaps, hardcoded shadows |
| LOW | 4 | Dead code, minor text inconsistencies |

---

## CRITICAL Issues (Fix Immediately)

### C1: ThemeController Creates New Instance Per Widget
- **File:** `lib/widgets/theme_toggle_widget.dart:10,103`
- **Issue:** `final themeController = ThemeController()` creates new instance each time
- **Fix:** Accept `ThemeController` via dependency injection
- **Impact:** Theme toggle does nothing - users cannot change themes

### C2: Hardcoded Orange Colors in Currency Banner
- **File:** `lib/screens/currency_converter_screen.dart:280-281`
- **Issue:** `Colors.orange.shade50` / `shade200` are hardcoded
- **Fix:** Use `theme.colorScheme.errorContainer` with `onErrorContainer`
- **Impact:** Colors break in dark mode and all non-default palettes

### C3: Hardcoded Green/Red in PurchaseButton
- **File:** `lib/widgets/purchase_button.dart:123,130,141,149,214,225`
- **Issue:** `Colors.green[50]`, `Colors.red[700]` are hardcoded
- **Fix:** Use `colorScheme.secondaryContainer` / `errorContainer`
- **Impact:** Status indicators break in dark mode and all palettes

### C4: Inconsistent Content Width Limits
- **File:** `currency_converter_screen.dart:217` vs `conversion_screen.dart:279`
- **Issue:** Currency uses 1180px, Conversion uses 1200px max width
- **Fix:** Standardize to 1200px across all screens
- **Impact:** Desktop layout inconsistent at same viewport widths

---

## HIGH Issues (Fix Before Next Release)

### H1: Inconsistent Card Widget Usage
- **Files:** `conversion_screen.dart` uses `Card.outlined`, `custom_units_screen.dart` uses `Card()`
- **Fix:** Standardize all cards to use `Card()` with theme's `CardThemeData` (elevation: 0 already set)

### H2: Mixed Button Styles in PurchaseButton
- **File:** `lib/widgets/purchase_button.dart:236-252`
- **Issue:** Uses `ElevatedButton` styled as `FilledButton`
- **Fix:** Use `FilledButton` directly

### H3: Responsive Layout Breakpoint Mismatch
- **File:** `responsive_layout.dart:9` (tablet=900px) vs `conversion_screen.dart:281` (wide=1000px)
- **Fix:** Use `ResponsiveLayout.getScreenSize()` consistently or same constant

### H4: Deprecated `primaryColor` Usage
- **File:** `purchase_button.dart:172,185,239`
- **Fix:** Replace with `colorScheme.primary`

---

## MEDIUM Issues (Nice to Have)

### M1: Inner Container Border Radius Mismatch
- **File:** `conversion_screen.dart:419`
- **Issue:** 16px inner radius vs 24px Card radius
- **Fix:** Standardize to 12px or 18px inner radius

### M2: Hardcoded Shadow in CurrencyScreen
- **File:** `currency_converter_screen.dart:323-326`
- **Fix:** Use `elevation` property or theme-aware shadows

### M3: Fixed Width Cards (Preset/Favorite/Recent)
- **Files:** `preset_card.dart:22` (188px), `favorite_conversion_card.dart:24` (220px), `recent_conversion_card.dart:26` (280px)
- **Fix:** Use `ResponsiveLayout.getHorizontalScrollColumns()` for responsive sizing

---

## LOW Issues (Polish)

### L1: Dead Class `SimpleThemeToggle`
- **File:** `theme_toggle_widget.dart:97-122`
- **Fix:** Remove unused class

### L2: Text Inconsistency ("value" vs "amount")
- **Files:** `unit_input_card.dart:65`, `currency_input_row.dart:59`
- **Fix:** Use "value" for unit conversions, "amount" for currency only

### L3: Duplicate Import
- **File:** `conversion_screen.dart:1-2`
- **Fix:** Remove duplicate `import 'dart:convert'`

### L4: Icon Size Mismatch in Category Card
- **File:** `category_card.dart:27,62`
- **Fix:** Balance icon container proportions with padding
