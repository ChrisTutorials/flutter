# Screenshot Fix Summary

## Problem
The store screenshots have:
1. White space at the bottom
2. Inaccurate cropping (top portion cut off)
3. Not showing the full app screen

## Root Cause
The raw screenshots in marketing/unit_converter/screenshots/raw_store_screenshots were not generated correctly. They had white space and were cropped.

## Solution Implemented

### 1. Updated Screenshot Spec
**File**: marketing/unit_converter/screenshots/store_screenshot_spec.json

Changed all parameters to ensure zero whitespace and no cropping:
- ackgroundTolerance: 0 - No tolerance for background color variation
- contentPadding: 0 - No padding around content
- maxBottomWhitespaceFraction: 0.0 - Zero whitespace allowed at bottom
- maxTopWhitespaceFraction: 0.0 - Zero whitespace allowed at top
- minCropScale: 1.0 - No cropping (use full image)
- ocusX: 0.5 and ocusY: 0.5 - Center focus

### 2. Created Screenshot Generation Test
**File**: 	est/integration/store_screenshots_generation_test.dart

This test generates full-device screenshots at the exact target dimensions:
- Phone: 1080x1920
- Tablet 7: 1200x1920
- Tablet 10: 1600x2560

The test:
1. Sets the device size to the target dimensions
2. Navigates to the required screens
3. Captures full-device screenshots
4. Saves them to the raw_screenshots directory

### 3. Created Screenshot Validation Test
**File**: 	est/integration/store_screenshots_validation_test.dart

This test ensures screenshots have:
1. Correct dimensions
2. Zero whitespace at bottom
3. Zero whitespace at top
4. Full app content (not cropped)

### 4. Added Image Package
**File**: pubspec.yaml

Added image: ^4.0.17 to dev_dependencies for screenshot validation.

## How to Use

### Generate New Screenshots

`ash

`

The processing step will now:
1. Copy raw screenshots to store_screenshots (no cropping since minCropScale=1.0)
2. Validate they meet the spec requirements
3. Fail if any whitespace is detected

## Troubleshooting

### If validation fails for whitespace
- The raw screenshots were not generated correctly
- Re-run the generation test
- Check that the device size is set correctly

### If validation fails for dimensions
- The device size was not set correctly in the generation test
- Check the setSurfaceSize calls in the generation test

### If validation fails for cropped content
- The screenshot capture is not capturing the full device
- Ensure setSurfaceSize is called before 	akeScreenshot
- Ensure surface size is reset after capture
