# Store Screenshot Requirements

This document is the source of truth for Google Play screenshot assets for this project.

## Orientation Rule

- Use portrait screenshots for phone, 7-inch tablet, and 10-inch tablet listings.
- The required aspect ratio for this project is 9:16.
- Do not prepare landscape screenshots for the main Play Store phone or tablet sets.

## File Format Rules

- Allowed formats: PNG or JPEG
- Maximum file size: 8 MB per image

## Phone Screenshots

- Upload count: 2 to 8 screenshots
- Minimum required by Play: at least 2 phone or tablet screenshots total
- Promotion eligibility target: include at least 4 screenshots with both dimensions at or above 1080 px
- Allowed dimensions per side: 320 px to 3840 px
- Required aspect ratio for this project: 9:16 portrait

Recommended project sizes:

- 1080 x 1920 for primary phone captures

## 7-Inch Tablet Screenshots

- Upload count: up to 8 screenshots
- Minimum required by Play: at least 2 phone or tablet screenshots total
- Allowed dimensions per side: 320 px to 3840 px
- Required aspect ratio for this project: 9:16 portrait

Recommended project sizes:

- 1200 x 1920 for 7-inch tablet captures

## 10-Inch Tablet Screenshots

- Upload count: up to 8 screenshots
- Minimum required by Play: at least 2 phone or tablet screenshots total
- Allowed dimensions per side: 1080 px to 7680 px
- Required aspect ratio for this project: 9:16 portrait

Recommended project sizes:

- 1600 x 2560 for 10-inch tablet captures

## Current Captured Set

The current exported files in docs/store_screenshots follow the portrait requirement:

- Phone: 1080 x 1920
- 7-inch tablet: 1200 x 1920
- 10-inch tablet: 1600 x 2560

## Content Priorities

Prefer screenshots that show the strongest differentiators first:

1. Home screen with search and category grid
2. Live conversion workflow
3. Custom units feature
4. Favorites, comparison, or formula surfaces
5. Settings and desktop/web-friendly layout where useful
