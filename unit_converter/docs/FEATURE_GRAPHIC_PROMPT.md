## Design Guidelines for Human Designer

If working with a designer, provide these guidelines:

### Color Palette
- Primary: Deep Purple (#6200EE)
- Secondary: Light Purple (#B388FF)
- Accent: Teal or Cyan for highlights
- Text: White (#FFFFFF) for contrast
- Background: Gradient from #6200EE to #3700B3

### Typography
- App Name: Bold, modern sans-serif (Roboto, Montserrat, or similar)
- Tagline: Lighter weight, smaller size
- Feature text: Clean, readable at small sizes

### Key Elements to Include
1. **App Name**: "Unit Converter" - prominent and legible
2. **Tagline**: "Convert Everything Instantly" or similar
3. **Visual Representation**: Phone mockup, icons, or conversion examples
4. **Feature Highlights**: 3-4 key features (10 Categories, Offline, Real-time, History)
5. **Measurement Icons**: Subtle decorative elements

### Layout Suggestions
- **Rule of Thirds**: Place key elements at intersection points
- **Visual Hierarchy**: App name > visual element > features
- **Breathing Room**: Don't crowd elements
- **Left-to-Right Flow**: Guide eye naturally

### What to Avoid
- Cluttered design with too many elements
- Low contrast text
- Generic stock photos
- Small text that won't be readable at 1024x500
- Dark backgrounds without sufficient contrast

### Export Requirements
- Dimensions: 1024 x 500 pixels
- Format: PNG (for transparency if needed) or JPEG
- Color profile: sRGB
- Resolution: 72 DPI (web standard)
- File size: Under 15 MB
